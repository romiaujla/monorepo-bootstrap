# UI Guidelines

## Purpose

This document provides lightweight UI consistency guidance for this bootstrap repository.

## Guidelines

- Prefer consistent spacing, typography, and action hierarchy across screens.
- Design loading, empty, error, and success states as first-class states.
- Reuse established layout and interaction patterns instead of inventing new ones without justification.
- Keep accessibility and responsive behavior in scope from the first wireframe.
- Favor components and tokens that can survive project templating and downstream customization.
